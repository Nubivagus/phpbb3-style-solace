Menus

Make sure that Devlom Configurator extension is installed in your phpBB. Otherwise changes made to files below won't be visible in your forum.

In Solace, we keep menus data in following YAML files located inside solace/content directory:

  • topmenu.yaml - menu data for top menu.
  • quickmenu.yaml - menu data for footer "Quick Menu" section.
  • socialmenu.yaml - footer social icons.
  • copyrightmenu.yaml - menu data for copyright section.

Top Menu

Let's take a look at topmenu.yaml. Below is a first menu item: 

- name: Home
  url: ./
  icon: home
  icon_provider: fontawesome
  active: true

As you can see, whole syntax is very easy to read and understand. Let's describe each part: 


- name: Home # Menu item name
  url: ./ # Menu item url (absolute or relative)
  icon: home # Icon name
  icon_provider: fontawesome # Icon provider: fontawesome or lineicon
  active: true # enable or disable active menu highlight

Icon and Icon Providers

Solace allows you to pick icon from two providers. FontAwesome nad ET-Line Icons (icon names in the middle of the page). In both cases, all you have to do is simply provide an icon name. So for example, if FontAwesome icon name is fa-bars then all you have to do is provide: 

icon: bars
If ET-Line Icon class name is .icon-chat then all you have to do is provide:

icon: chat

Ready to add another menu item ? It's easy.


- name: Another menu item
  url: "http://www.devlom.com"
  icon: desktop
  icon_provider: lineicon
  active: true

How easy is that ? :) The whole code for two menu items should look like that:


- name: Home
  url: ./
  icon: home
  icon_provider: fontawesome
  active: true
- name: Another menu item
  url: "http://www.devlom.com"
  icon: desktop
  icon_provider: lineicon
  active: true

Adding submenu

Adding submenu is also very easy. Let's add submenu to our second item. Whole code will look like that:


- name: Home
  url: ./
  icon: home
  icon_provider: fontawesome
  active: true
- name: Another menu item
  url: "http://www.devlom.com"
  icon: desktop
  icon_provider: lineicon
  active: true
  submenu:
    - name: Icon Item
      url: "#"
      icon: bank
      icon_provider: fontawesome
      active: true
    - name: Example Item
      url: "#"
      icon:
      icon_provider:
      active: false
As you can see from above example, key value is called submenu: which is also a container. You can add as many menu items to each container as you want. There's one limitation though. Solace supports maximum 4 level menu items.

topmenu.yaml is a perfect example to look if you want to learn how to create complex submenus.

Quick menu

Quick menu is located in forum footer. Quick menu syntax is almost the same as Top Menu except that you can't create submenus and highlight active item. Example menu items:

- name: Example Item 1
  url: "#"
- name: Example Item 1
  url: "#"

quickmenu.yaml is a perfect example to look if you want to learn how it works

Copyright menu

Quick menu is located in forum copyright section. Below footer. Quick menu syntax is very basic. You can specify name and url only. Example menu items:

- name: Example Item 1
  url: "#"
- name: Example Item 1
  url: "#"

copyrightmenu.yaml is a perfect example to look if you want to learn how it works

Social menu

Social menu is an addition to footer logo module. For social menu you can specify icon, icon provider and url. Example:

- icon: facebook
  url: "#"
  provider: lineicon
- icon: twitter
  url: "#"
  provider: lineicon
- icon: linkedin
  url: "#"
  provider: lineicon

socialmenu.yaml is a perfect example to look if you want to learn how it works
